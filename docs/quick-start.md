# 禅道月度工作整理快速使用说明

这套工具可以结合 AI 对话、Git 提交和人工工作描述，整理月度需求、任务、Bug，并在确认后上传禅道。

## 一、让 Codex 自动安装

打开一个 Codex 会话，把下面内容发给 Codex，并替换其中的个人信息：

```text
请帮我安装并配置禅道月度工作整理工具。
代码仓库：git@github.com:Max0314/zentao-monthly-worklog.git

请完成以下操作：
1. 将仓库克隆到合适的独立目录。
2. 检查 Python 3.10+ 和 Git，执行 python -m pip install -e .。
3. 安装 zentao-monthly-worklog Skill：Windows 执行 scripts/install_skill.ps1；Linux/macOS 将 skills/zentao-monthly-worklog 复制到 ~/.codex/skills/。
4. 使用以下信息完成首次配置：
   禅道账号：你的账号
   禅道密码：你的密码
   代码工作区：D:\你的代码目录
   Git 作者：你的姓名、你的 Git 邮箱
5. 使用正式禅道，验证连接并列出我可访问的产品、项目和迭代。

不要创建或修改任何禅道数据。安装完成后告诉我配置文件位置和验证结果。
```

安装完成后，重新打开一个 Codex 会话，让新安装的 Skill 生效。

## 二、推荐提示词一：生成月度清单

日常使用时，建议先生成清单确认，不直接上传：

```text
使用 $zentao-monthly-worklog 分析 2026 年 7 月的 AI 对话和 Git 变更。
结合各项目实际提交、测试和部署记录，整理需求、任务和 Bug，并分类到对应迭代。
每项包含详细描述和逐条独立备注；区分已完成事项与待确认候选，不能把只有 fix 提交但缺少问题和验证证据的内容直接当作已解决 Bug。
先给我完整清单确认，不要上传禅道。
```

把年月替换成需要整理的月份即可。

## 三、推荐提示词二：上传已确认清单

检查并确认清单后，再发送：

```text
使用 $zentao-monthly-worklog 将刚才确认的完整清单上传正式禅道。
按已确认的产品、项目和迭代创建需求、任务和 Bug。
每条任务完成步骤、Bug 解决步骤和伪代码都必须作为单独备注写入；完成任务并解决 Bug。
上传后等待 AI 评分，回查标题、归属、状态、备注数量和评分，并列出全部真实禅道 ID。
如发现同标题记录或环境不一致，停止处理并告诉我，不要自动接管或重复创建。
```

## 四、没有 AI 对话时

仍然可以结合 Git 和工作描述生成候选清单：

```text
使用 $zentao-monthly-worklog 整理 2026 年 7 月工作。
本月没有可用的 AI 对话，请结合 Git 变更和以下工作说明生成需求、任务和 Bug 候选：这里填写本月工作内容。
缺少完成或修复证据的事项标记为“待确认”，不要虚构已完成任务或已解决 Bug，不要上传禅道。
```

## 五、注意事项

- 首次使用必须确认产品、项目和迭代归属。
- 默认先看清单，再明确授权上传。
- 禅道账号配置保存在当前用户目录，不应提交到业务仓库。
- 上传中断后应复用原 Manifest 继续，避免重复创建和重复备注。
- NeoCoder 等工具可以直接调用 Python 工具；若不兼容 Codex Skill 规范，则不能使用 `$zentao-monthly-worklog` 名称自动触发。

完整说明见 [trial-guide.md](trial-guide.md)。
