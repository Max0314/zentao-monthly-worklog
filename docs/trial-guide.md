# 禅道月度工作整理工具试用说明

这套工具会读取你的 AI 编码对话、Git 提交和人工工作说明，整理出月度需求、任务、Bug 及逐条备注。默认先生成清单给你确认，只有明确同意后才上传禅道。

## 一、准备环境

电脑需要安装：

- Python 3.10 或更高版本
- Git
- Codex 桌面端或其他能够执行命令的 Agent
- 能访问公司正式禅道内网或外网

安装命令：

```powershell
git clone git@github.com:Max0314/zentao-monthly-worklog.git
cd zentao-monthly-worklog
python -m pip install -e .
powershell -ExecutionPolicy Bypass -File .\scripts\install_skill.ps1
```

安装完成后，重新打开一个 Codex 会话。

## 二、首次配置

把下面文字发给 Codex，并替换成自己的信息：

```text
使用 $zentao-monthly-worklog 完成首次配置。
禅道账号：你的账号
禅道密码：你的密码
代码工作区：D:\你的代码目录
Git 作者：你的姓名、你的Git邮箱
使用正式禅道，先验证连接并列出我可以访问的迭代，不要创建任何数据。
```

配置保存在当前用户目录的 `~/.zentao-monthly-worklog/config.local.json`，不会写入业务代码仓库。正式环境默认先访问内网，内网不可达时自动使用外网地址。

Codex 会列出可能属于你的迭代。请认真确认产品、项目和迭代，不要直接沿用其他同事的配置。

## 三、生成月度清单

推荐先分析，不上传：

```text
使用 $zentao-monthly-worklog 分析 2026 年 7 月的 AI 对话和 Git 变更。
结合实际提交、测试和部署记录，整理需求、任务和 Bug，并分类到对应迭代。
每项包含详细描述和逐条独立备注；区分已完成事项与待确认候选。
先给我完整清单确认，不要上传禅道。
```

工具会自动处理：

- 跨月持续的 Codex 会话按每条消息的实际月份归类。
- 同一 Git 提交出现在多个 worktree 时只统计一次。
- `fix:` 提交先作为 Bug 候选，证据不足时仍可作为任务，不会直接消失。
- 没有问题表象、修复过程和验证结果的内容不会冒充已解决 Bug。

清单生成后，请重点确认标题、迭代归属、完成状态和是否遗漏重要项目。

## 四、上传测试禅道

首次使用建议先上传少量测试数据：

```text
从已确认清单中各选 2 条需求、任务和 Bug 上传测试禅道。
每条完成步骤、解决步骤和伪代码都作为独立备注。
完成任务、解决 Bug，并回查标题、状态、备注数量和 AI 评分。
正式禅道不要写入。
```

测试上传成功后，结果中应包含真实禅道 ID、任务状态 `done`、Bug 状态 `resolved/fixed`、备注匹配数量和评分数量。

## 五、上传正式禅道

确认完整清单和测试结果后再发送：

```text
按刚才确认的完整清单上传正式禅道。
使用 formal_auto 自动选择内外网地址，逐条写入备注，完成任务并解决 Bug。
上传后等待备注 AI 评分，并回查标题、状态、备注和评分，最后列出真实 ID。
```

工具会要求输入大写 `YES` 才开始写入。Agent 已获得本次完整清单明确授权时，可以使用自动确认参数。

## 六、中断后继续

网络中断或禅道临时失败时，不要删除 Manifest，也不要重新生成一套相同标题的数据。发送：

```text
继续上次月度禅道上传。复用原 Manifest，先按服务端真实备注核对进度，再继续上传，禁止重复创建和重复备注。
```

Manifest 不能在测试和正式禅道之间混用，也不能换账号使用。记录内容在创建后发生变化时，工具会停止并要求人工确认。

## 七、同标题冲突

发现禅道已有同标题记录时，工具默认停止，不会自动修改已有记录。先打开禅道确认该记录确实属于你和本次清单，再明确告诉 Agent 接管该记录。Agent 会使用 `--adopt-existing`，补充缺少的备注并设置最终状态。

不要接管标题相同但属于其他人、其他月份或其他业务范围的记录。

## 八、没有 Codex 对话怎么办

可以结合 Git 和部门职责生成候选清单：

```text
使用 $zentao-monthly-worklog 整理 2026 年 7 月工作。
本月没有可用的 Codex 对话，请结合 Git 变更和以下工作说明生成候选清单：……。
缺少完成或修复证据的内容标记“待确认”，不要虚构已完成任务或已解决 Bug，不要上传禅道。
```

NeoCoder 等其他工具可以导出 Markdown、JSON 或 JSONL 后作为补充证据。

## 九、常见问题

### 找不到我的迭代

让 Agent 使用 `list-scopes --limit 1000` 重新查询。如果测试和正式禅道 ID 不同，需要分别配置环境映射。

### 提示同标题记录已存在

这是保护机制。先确认记录所有权，再决定接管；不要为了绕过保护随意修改标题或新建重复记录。

### 上传中断了

保留原 Manifest，直接要求继续。工具会读取服务端真实备注后恢复，不会只相信本地计数。

### 备注存在但没有评分

AI 评分可能有短暂延迟。让 Agent 使用带等待参数的验证命令重新检查，不需要重复提交备注。

### 只想看清单，不想上传

在提示词结尾明确写“先给我完整清单确认，不要上传禅道”。这是最推荐的日常使用方式。
